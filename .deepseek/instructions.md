# Economic Analysis Research — Project Instructions

> Локальный аналитический проект: экономические исследования на данных МВФ (IMF SDMX API)
> и других источников. Страны, индикаторы, периоды — любые комбинации.
> Материалы: статьи, презентации, дашборды, промпты для AI-агентов.
> MCP-сервера подключаются локально, без внешних API-ключей.

## MCP Servers

### IMF Data (`imf-data-mcp` v0.2.0)
- **Бинарник:** `.uv-bin/imf-data-mcp`
- **Источник:** `github.com/c-cf/imf-data-mcp` (обёртка над `imfp` → IMF SDMX 3.0 API)
- **API-ключ:** не требуется
- **Лимиты:** ~10 запросов / 5 сек (авто-ретрай через `imfp`)
- **Инструменты:**
  - `imf_list_databases` — все ~155 баз IMF
  - `imf_search_databases` — поиск по ключевым словам
  - `imf_get_parameter_defs` — измерения базы (country, frequency, indicator...)
  - `imf_get_parameter_codes` — коды измерений с поиском
  - `imf_fetch_data` — временные ряды с фильтрами
- **Ключевые базы для проекта:**
  | Тема | ID |
  |---|---|
  | Потребительские цены | `CPI` |
  | Цены производителей | `PPI` |
  | Национальные счета (год) | `ANEA` |
  | Национальные счета (квартал) | `QNEA` |
  | Платёжный баланс | `BOP_AGG` |
  | Валютные курсы | `ER_2026_JAN_VINTAGE` |
  | Сырьевые цены | `PCPS` |
  | Фискальный монитор | `FM` |
  | Госфинансы | `GFS_COFOG` |
  | Монетарная статистика | `MFS_CBS`, `MFS_MA`, `MFS_IR` |
  | Финансовая устойчивость | `FSIC` |
  | Торговля услугами | `ITS` |

## Поведенческие правила (метода Карпати)

**Приоритет: осторожность выше скорости. Для тривиальных задач — по ситуации.**

### 1. Думай перед кодом
- **Не додумывай. Не прячь неуверенность. Показывай альтернативы.**
- Перед реализацией:
  - Проговори допущения явно. Если неуверен — спроси.
  - Если есть несколько интерпретаций — представь их, не выбирай молча.
  - Если есть более простой подход — скажи. Спорь, когда это оправдано.
  - Если что-то неясно — остановись. Назови, что смущает. Спроси.

### 2. Сначала простота
- **Минимальный код, решающий задачу. Ничего speculative.**
  - Никаких фич сверх запрошенного.
  - Никаких абстракций для одноразового кода.
  - Никакой «гибкости» или «конфигурируемости», которую не просили.
  - Никакой обработки ошибок для невозможных сценариев.
  - Если написал 200 строк, а можно 50 — перепиши.
- Тест: «Назвал бы senior-инженер это переусложнённым?» Если да — упрощай.

### 3. Хирургические правки
- **Трогай только необходимое. Убирай только свой мусор.**
  - Не «улучшай» соседний код, комментарии или форматирование.
  - Не рефакторь то, что не сломано.
  - Соблюдай существующий стиль, даже если сделал бы иначе.
  - Если заметил мёртвый код — упомяни, не удаляй.
- Когда твои изменения создают сирот:
  - Удали импорты/переменные/функции, которые стали неиспользуемыми из-за ТВОИХ правок.
  - Не удаляй ранее существовавший мёртвый код без запроса.
- Правило: каждая изменённая строка должна вести к запросу пользователя.

### 4. Целе-ориентированное исполнение
- **Определи критерии успеха. Зацикливайся до верификации.**
  - «Добавь валидацию» → «Напиши тесты на невалидный ввод, затем сделай чтобы проходили»
  - «Почини баг» → «Напиши тест, воспроизводящий баг, затем почини»
  - «Отрендерь X» → «Проверь: файл существует, страниц ≥ N, текст извлёкся»
- Для многошаговых задач — краткий план:
  ```
  1. [Шаг] → проверить: [критерий]
  2. [Шаг] → проверить: [критерий]
  ```
- Сильные критерии успеха позволяют работать автономно. Слабые («сделай чтоб работало») требуют постоянных уточнений.

## Рабочие процессы

### Типовой аналитический цикл
```
1. Поиск базы:     imf_search_databases(keyword="...")
2. Изучение структуры: imf_get_parameter_defs(database_id="...")
3. Поиск кодов:    imf_get_parameter_codes(database_id="...", parameter="country", search="<страна>")
4. Загрузка данных: imf_fetch_data(database_id="...", start_year=..., end_year=..., filters={...})
5. Анализ:         Python (pandas) → графики, статистика
6. Вывод:          текст статьи / слайды презентации / промпт для другого агента
```

### Типы deliverables
- **Статья** — Markdown в `articles/`, с графиками (сохранять в `charts/`)
- **Презентация** — PPTX через `python-pptx` в `presentations/`
- **Промпт для агента** — структурированный промпт в `prompts/` для делегирования подзадач
- **Датасет** — CSV/Parquet в `data/` с документированием источника и фильтров
- **Дашборд** — HTML/React в `dashboard/` (при необходимости)

### Конвенции файлов
```

```
